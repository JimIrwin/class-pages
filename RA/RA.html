 Coding instructions 

To learn coding, review these coding instructions; this folder has examples (Albany volume 163 spreadsheet & deed images;  Hustings (Richmond) DB 38 spreadsheet & deed images)

For all sampled deeds, code this subset of variables:  [ page date FPtype FPnames FPresid  illit   value cu deed  SPtype  fem ].  This is called "illiteracy coding."    For sale deeds only, code all variables ("sales coding").  If value=$1 or 6 shillings then it is not a "sales deed", code deed variable as nominal.

Done=1 When coding, be alert for rows with that have been completed; these have the number 1 in column A (the variable named "done"); do not modify such rows, and do not enter anything in the "done" column (I use the column to keep track of the status of a record).  

2Ptpe  and  fem.  Variable fem=1 if a party of the first or second parts is a woman acting without a husband -- usually ptype of "f" but possibly ptype of "W" if her husband is not on the deed with her).  Do not spend much time on these variables. If they are not obvious in any one deed then code u for 2Ptype and/or code 5 for fem. 

 assorted fine points for coding;

use W for "wife" and not "f" (female) when a party to a deed is identified as a wife -- including cases where the wife is acting alone, such as buying property (an in that case, code fem=1) for FPtype and SPtype use  D for widow  and R for widower


code occupations in parentheses after names  Smith albert (blacksmith)  

use abbreviations for location names ...  In general, use only the first 4-6 letters in town names and county names Â– T.Beth for Bethlehem,   T.Guild  Guilderland,  Rensseleaer  T.Renss      Watervliet T.Waterv    T.Coey for Coeymans     For counties, use 3-7 letters, such as Co.Alb   Co.Renss  Co.Berksh   Co.Oneida  Co.Schen   Co.Onond   Co.Saratog .    The exact abbreviation is not important, but do avoid unnecessary length.   Location coding:  when coding location be careful with prefixes Co (county)  Ci (city)  T (town or township)   V (village)  -- typically you use those prefixes only if they used in the original. For example, a deed with  "Jim Irwin of West Troy"  selling land "in the village of West Troy"  -- 1Pres is WTroy ;  loc is V.WTroy     Similarly, in Worcester deeds you might see  "Jim Irwin of the County of Worcester" (Co.Worc)  or "Jim Irwin of the City of Worcester in the County of Worcester" (Ci.Worc)  or "Jim Irwin of Worcester in the County of Worcester" (Worc -- no prefix).

Use IMAGE NUMBERS to name image files that you download, put those files in a folder that is named to identify the source of the images. Be careful that the image number is correct. 
* To get download images to Professor Irwin, consider using your "OneDrive" at CMU: put the images into a zipped folder and upload the resulting zip-file to your OneDrive; then from the "Share" menu for that file you can get a link to that file and email it to Dr Irwin.  Tim has done this, in case you have questions (or go see Dr Irwin for a demonstration).

square footage (sq ft or  lot size ) enter the dimensions as a calculation in the spreadsheet cell.  If a lot is 100' x 30' enter =100*30 and do not enter 3000 (but 3000 will be displayed).

Money in Pounds :     One Hundred Pounds 6/8    means L 100 6 shillings  8 pence   = (100+6/20+8/240) =100.3333   Notice the  /   separates shillings and pence   


* coding tips -- all for economizing on characters to be stored in the database:
  don't repeat surnames in a cell -- instead of : Osborn william & ann amelia, Osborn ransom & amanda, Osborn jesse
                                                            code: Osborn william & ann amelia, ransom & amanda, jesse
  abbreviate compass directions as  N E S W  NW  SW  etc (don't write out south or southerly, etc)
   abbreviate West Troy  as WTroy
  property names -- skip the word "number" for lot names.  Code Lot 34  not  Lot number 34 (nor Lot no. 34) -- but street addresses may be coded with num  (e.g. num 269) for lot known as Number 269 on Dove St   [road: Dove St   name: num 269]
